一、什么是AOP
AOP即面向切面编程,是对传统OOP的补充。AOP的主要编程对象是切面。

二、AOP中的一些概念
横切关注点(cross-cutting concern):散布于应用程序中多处的功能(需求)被称
为横切关注点从概念上来看是与业务逻辑相分离的。把横切关注点与业务逻辑代码分离
开来是面向切面编程所要解决的问题。
切面(aspect):横切关注点(一些共同需求)被模块化之后的特殊对象(通知和切点的结合)
通知(advice):切面是什么(切面必须完成的功能),切面什么时候被使用
目标(target):被通知的对象
代理(proxy):对目标对象应用通知后
连接点(JoinPoint):应用程序有数以千计的时间可以被应用通知，这些时机被称为连接点
切点(pointcut): 一个切面并不需要通知所有的连接点，切点缩小了所需通知的连接点的范围

三、使用AOP的好处:
1.横切关注点被集中于一个地方(切面)，而不是分散到多处代码中
2.服务模块更简洁，服务模块只需关注与业务逻辑相关的代码

四、Spring对AOP的支持
基于代理的经典AOP
纯POJO切面(基于XML，将纯POJO转换成切面)
@AspectJ注解驱动的切面(通过AspectJ提供注解驱动的AOP，本质上仍然是Spring基于代理的AOP)
注入式AspectJ注解驱动

前三种都是Spring AOP实现的变体，Spring AOP构建在动态代理的基础之上。因此Spring对AOP的支持仅限于方法拦截
如果要实现更高的需求,如构造器或者属性拦截，则需要使用AspectJ来实现切面，即注入式AspectJ注解驱动

五、Spring中@AspectJ注解驱动的切面

基本步骤
1.通过切点来选择连接点(使用AspectJ切点表达式语言)
AspectJ指示器execution()：用于匹配是连接点的方法

切点表达式的使用规则：
execution(modifiers-pattern? ret-type-pattern declaring-type-pattern? name-pattern(param-pattern) throws-pattern?)

有“?”号的部分表示可省略的，modifers-pattern表示修饰符如public、protected等，ret-type-pattern表示方法返回类型，declaring-type-pattern代表特定的类，name-pattern代表方法名称，param-pattern表示参数，throws-pattern表示抛出的异常。在切入点表达式中，可以使用*来代表任意字符，用..来表示任意个参数。

2.创建切面
@Aspect声明切面

五种通知
@Before 通知方法会在目标方法执行之前执行
@After 通知方法会在目标方法执行之后执行(无论目标是否发生异常)
@AfterReturning 通知方法会在目标方法正常返回之后执行
@AfterThrowing 通知方法会在目标方法抛出异常之后执行
@Around 通知方法会将目标方法包裹(封装)起来

@order 声明切面的优先级
@pointcut 声明可重用的切点

3.启用自动代理功能
在xml文件中使用aop命名空间中的<aop:aspectj-autoproxy />标签启用自动代理(或者使用JavaConfig)

六、注解驱动的切面的具体实现:

public interface Calculator {

    public double add(double i,double j);

    public double sub(double i,double j);

    public double mul(double i,double j);

    public double div(double i,double j);
}

import org.springframework.stereotype.Component;

@Component
public class CalculatorImp implements Calculator {

    @Override
    public double add(double i, double j) {
        double result = i + j;
        return result;
    }

    @Override
    public double sub(double i, double j) {
        double result = i - j;
        return result;
    }

    @Override
    public double mul(double i, double j) {
        double result = i * j;
        return result;
    }

    @Override
    public double div(double i, double j) {
        double result = i / j;
        return result;
    }

    }

声明一个日志切面

import java.util.Arrays;
import java.util.List;

import org.aspectj.lang.JoinPoint;
import org.aspectj.lang.ProceedingJoinPoint;
import org.aspectj.lang.annotation.After;
import org.aspectj.lang.annotation.AfterReturning;
import org.aspectj.lang.annotation.AfterThrowing;
import org.aspectj.lang.annotation.Around;
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Before;
import org.aspectj.lang.annotation.Pointcut;
import org.springframework.stereotype.Component;

@Component
@Aspect // 声明位一个切面类
public class LoggingAspect {

    @Pointcut("execution(public double  com.jzf.spring.aop.CalculatorImp.*(double,double))")
    public void pointcut() {
    }

    // 声明前置通知方法(目标方法执行之前执行)
    @Before("pointcut()")
    public void beforeMethod(JoinPoint jp) {
        String methodName = jp.getSignature().getName();
        List<Object> list = (List<Object>) Arrays.asList(jp.getArgs());// 得到方法的参数
        System.out.println(methodName + " begin with" + list);
    }

    // 后置通知方法(目标方法执行之后执行，无论方法是否出现异常;无法得到返回值,因为可能抛出异常)
    @After("pointcut()")
    public void afterMethod(JoinPoint jp) {
        String methodName = jp.getSignature().getName();
        System.out.println(methodName + " end");
    }

    //返回通知方法
    @AfterReturning(value="pointcut()",returning="result")
    public void afterReturning(JoinPoint jp,double result){
        String methodName = jp.getSignature().getName();
        System.out.println(methodName+" return with:"+result);
    }

    //异常通知方法
    @AfterThrowing(throwing = "ex", value = "pointcut()")
    public void throwException(JoinPoint jp, Exception ex) {
        String methodName = jp.getSignature().getName();
        System.out.println(methodName + " occurs exception: " + ex);
    }

}
