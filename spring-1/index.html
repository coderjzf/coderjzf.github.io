一、Spring简述
Spring是一个开源框架，是为简化企业级应用开发而生。Spring是一个IoC(DI)和AOP框架。

Spring为简化Java开发的复杂性，采取了以下四种关键策略：
基于POJO的轻量级和最小侵入性编程
通过依赖注入和面向接口实现松耦合

基于切面和惯例进行声明式编程
通过切面和模板减少样板式代码

在Spring4.0中，包括了20个不同的模块。

  • Spring核心容器：Spring最核心的部分，管理者bean的创建、配置和管理。Spring bean工厂提供DI的功能，而Soring中的应用上
    下文是在bean 工厂的基础上实现的。
  • Spring AOP模块:Spring中开发切面的基础。借助于AOP,可以将遍布于系统中的横切关注点从它们应用的对象中解耦出来。
  • 数据访问与集成(ORM模块)
  • Web与远程调用(Spring MVC)
  • …

二、IoC&DI
IoC:控制反转 反转资源获取的方向
传统的资源查找要求组件向容器发起请求来查找资源，容器适时地返回资源，而应用了IoC之后容器主动将资源推送给它所管理的组件，
组件所要做的仅是选择一种合适的方式来接受资源

DI:依赖注入 IoC的另一种表述形式
组件以一些方式(如构造器、setter方法)接受来自容器的资源注入

Spring IoC容器的两种实现方式
1.BeanFactory
2.ApplicationContext接口(基于BeanFactory)
ClassPathXmlApplication 从类路径下加载配置文件
FileSystemXmlApplication 从文件系统中读取配置文件
..

配置bean
1.基于xml文件的方式

依赖注入的方式
属性注入(即setter方法注入，最常用的方式)
构造器注入 <constructor-arg>

每种注入方式又包括字面量注入，引用注入，集合注入，Map注入等

bean之间的关系 继承、依赖
继承：parent属性指定要继承的bean
依赖：depends-on属性指定依赖的bean

bean的作用域
使用bean的scope属性为bean设置作用域 默认值：singleton
singleton(单例的):在容器被创建时就初始化bean，每次请求返回到都是同一个bean实例
prototype(原型的):在容器被创建时没有初始化bean,每次请求都初始化并返回一个bean实例

使用外部属性文件

  1. 导入context命名空间
  2. 配置properties文件
  3. 使用<context:property-placeholder/> 导入properties文件
  4. 使用属性占位符${}取出properties中的值

SpEL Spring表达式语言#{}

  1. 表示字面量
  2. 引用bean、属性和方法 ？来判断是非为null值
  3. 使用Java中的类型
  4. SpEL运算符(算术、比较、逻辑、条件) 三元运算符来判断是否为null值
  5. SpEL通过matches支持正则表达式
  6. SpEL操作集合

Spring IoC容器可以管理bean的生命周期
bean的生命周期过程：

  1. 通过构造器或者工厂方法创建bean实例
  2. 为bean的属性赋值
  3. 将bean实例传给bean的后置处理器的postProcessBeforeInitialization方法
  4. 调用bean的初始化方法
  5. 将bean实例传给bean的后置处理器的postProcessAfterInitialization方法
  6. bean可以使用了
  7. 当容器关闭时，调用bean的销毁方法

3,5为可选的
可以通过实现BeanPostProcess接口，来自定义bean的后置处理器，在bean的初始化方法调用的前后对bean进行处理

2.通过注解配置bean
常用注解
@Component

组件扫描：容器从classpath下自动扫描具有特定注解的组件
<context:component-scan base-package=""> base-package指定要扫描的基类包
resource-pattern属性指定扫描基类包下的某个包/类
<context:include-filter>指定包含要扫描的某个包
<context:exclude-filter>排除某个包

autowired可以自动装配类型兼容的bean
可以放在普通字段、构造器等一切有参数的方法上
