Spring(二):IoC&DI

发表于 2017-06-16 | 分类于 JavaWeb

一、IoC&DI是什么

1.1 IoC:Inverse Of Control(反转控制)：

  • 传统的对象创建方式：对象的创建都是由程序员自己开发，对象之间的依赖关系也是由程序员来注入。
  • IoC:反转了对象的创建方式，对象的创建与对象之间的依赖的关系都交由Spring容器管理。

1.2 DI:Dependency Injection(依赖注入)：

为了实现IoC需要DI的支持，对象的创建(IoC)需要依赖注入。

注入方式：

  • set方法注入
  • 构造函数注入
  • P命名空间&SpEL注入

注入类型：

  • 值类型注入:Java简单类型的注入
  • 引用类型注入

二、BeanFactory&ApplicationContext

2.1 容器对象：

  • BeanFactory:最底层的接口，针对这个接口的实现类功能较为单一。BeanFactory接口实现类的容器，在每次获取对象时才会创建对象。
  • ApplicationContext:这个接口实现类的容器在容器对象创建时，所有容器中的对象都会被创建，而且功能更为强大。
    • ClassXmlPathApplicationContext:ApplicationContext的实现类，从类路径下加载配置文件来创建容器。
    • FileSystemXmlApplicationContext:ApplicationContext的实现类,从硬盘文件中加载配置文件来创建容器。

应用场景：一般使用ApplicationContext,在资源缺乏的环境下可以使用BeanFactory。

3.2 scope属性

scope属性用来指定创建对象的模式：

  • singleton:默认值，即单例对象，容器对象被创建时即存在。容器中只会存在一个对象实例
  • prototype:多例原型对象，每次从容器中获取对象时，对象才会被创建。而且每次都创建一个新的对象实例
  • request:对象与Web中request的生命周期一致
  • session:与Web中session的生命周期一致

3.3 生命周期属性

init-method:指定一个初始化方法，容器会在对象创建时调用
destroy-method:指定一个销毁方法，容器会在对象销毁时调用

4.3 复杂类型属性的注入

复杂类型如：

  • Array
  • List
  • Map
  • Properties

4.3.1 示例：

4.3.1.1 CollectionBean实体类
import java.util.Arrays;
import java.util.List;
import java.util.Map;
import java.util.Properties;

public class CollectionBean {
    private Object[] array;// 数组类型属性的注入
    private List list; // List类型属性的注入
    private Map map; // Map类型属性的注入
    private Properties proerties; // Properties类型属性的注入

    public Object[] getArray() {
        return array;
    }

    public void setArray(Object[] array) {
        this.array = array;
    }

    public List getList() {
        return list;
    }

    public void setList(List list) {
        this.list = list;
    }

    public Map getMap() {
        return map;
    }

    public void setMap(Map map) {
        this.map = map;
    }

    public Properties getProerties() {
        return proerties;
    }

    public void setProerties(Properties proerties) {
        this.proerties = proerties;
    }

    @Override
    public String toString() {
        return "CollectionBean [array=" + Arrays.toString(array) + ", list=" + list + ", map=" + map + ", proerties="
                + proerties + "]";
    }
}
4.3.1.2 配置文件applicationContext.xml
<bean name="collectionBean" class="jzf.spring.inject.CollectionBean">
    <!-- 如果数组中只存放一个值，可以直接用value/ref子属性来配置 -->
    <!-- <property name="array" value="Mike"></property> -->
    <!-- 数组类型注入：
         array:包含数组的元素
         value:数组元素
         ref:引用其他类型作为数组的元素
     -->
    <property name="array">
        <array>
            <value>Mike</value>
            <value>Jack</value>
            <ref bean="user" />
        </array>
    </property>

    <!-- 如果List中只存放一个值，也可以直接用value/ref子属性来配置 -->
    <property name="list">
        <list>
            <value>Rose</value>
            <ref bean="user" />
        </list>
    </property>

    <!--Map类型的注入： 
        entry:map中的键值对 
        key:键
        key-ref:引用其他类型作为键 
        value:值 
        value-ref:引用其他类型作为值 -->
    <property name="map">
        <map>
            <entry key="username" value="root"></entry>
            <entry key="u" value-ref="user"></entry>
        </map>
    </property>

    <!-- Proerties类型注入
         props:Proerties中的元素必须包含在其中
         key:键
         prop:标签体内即为值
    -->
    <property name="proerties">
        <props>
            <prop key="driver">com.mysql.jdbc.Driver</prop>
            <prop key="url">jdbc:mysql:///test</prop>
        </props>
    </property>
</bean>
